# Benchmark datasets (`benchmark/data/`)

Per the engineering-review P3, the benchmark needs reproducible image tiers:

| Tier    | Images (per format) | Approx size on disk |
| ------- | ------------------- | ------------------- |
| small   | 100                 | ~20 MB              |
| medium  | 1000                | ~200 MB             |
| large   | 10000               | ~2 GB               |

Each tier contains JPEG + PNG (512×512) and TIFF (512×512) images generated by
the deterministic corpus generator inside `mviewer_bench` (gradient + noise, so
decoders do real work — no flat-color shortcut).

## Generate

```powershell
# from repo root, after a Release build
powershell -ExecutionPolicy Bypass -File benchmark/generate_bench_data.ps1
```

By default this materializes `small` + `medium` into `D:/mviewer_bench_data/`
(to avoid starving the C: system drive). Add `large` (or `-Tiers all`) to also
generate the 10000-image tier (~2 GB). Output location is configurable via
`-OutRoot`.

## Use a tier as the benchmark corpus

The benchmark harness writes its runtime corpus to `MVIEWER_BENCH_TMP`. Point it
at a generated tier to benchmark against a fixed, reproducible dataset:

```powershell
$env:MVIEWER_BENCH_TMP = "D:/mviewer_bench_data/medium"
./build_msvc/bin/mviewer_bench.exe --corpus-size 1000 --enforce
```

## Note

The images are **not committed to git** (binary bloat). They are regenerated on
demand by `generate_bench_data.ps1`, which reuses the already-tested
`makeCorpus` path in `src/benchmark/corpus.cpp`. TIFF was reduced from
2000×2000 to 512×512 (commit `d2d56c7`) so the `large` tier fits on a data disk
instead of consuming ~12 GB.
