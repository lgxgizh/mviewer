# Milestone M3 — Core Image Pipeline (Frozen Baseline)

**Status:** Frozen — `✅ Done` (Phase-1 + Phase-2 + cleanup; see Phase-3 note)

**Purpose of this doc:** Record the M3 acceptance baseline so later milestones

## Goal

A production image pipeline where UI never decodes. All image access flows
through `ImageRepository`.

## Architecture (frozen)

```
UI (Qt Widgets)
   │  (no fopen / no QImageReader / no decode)
   ▼
Application (UseCases)
   ▼
Core: ImageRepository → Decoder(Registry) → Cache(LRU) → ImageFrame
   ▼
Domain: ImageFrame, Selection, CompareSession  (Qt-free)
```

## Implemented & Verified (M3 Phase-1 + Phase-2)

- `ImageRepository::load(path)` returns an `ImageFrame` (pixels + metadata +
  decode state + histogram placeholder).
- `ImageRepository::loadDirectoryAsync` — non-blocking parallel load; returns
  the file list immediately, decodes on `DecodePool`, delivers completion via
  `TaskScheduler::drain()` (not `processEvents()`). **Verified:** 1000 images
  return in ~15 ms (budget ≤100 ms); all 1000 frames decode via the async path.
- Formats: **JPEG / PNG / BMP** decode through `DecoderRegistry` → `QtDecoder`.
  **TIFF** is listed by the pipeline; decode requires `qtiff` plugin +
  MSVC-built `libtiff-6.dll` deployed beside the executable (codec-gated test
  auto-skips when absent).
- In-memory Viewer/FullImage **LRU** cache; adjacent-image switch is instant
  after first decode (second load served from LRU, identical pixels, no disk
  round-trip).
- Background **ThumbnailPipeline**: directory scan → immediate file list →
  background thumbnail generation → LRU → UI update. **Verified:** first
  thumbnail in ~3 ms (budget ≤200 ms).
- **Pixel Inspector**: `ImageViewer` emits `pixelInfo(x,y,r,g,b,valid)` on
  mouse move, read directly from `ImageFrame` pixels, wired to the main-window
  status bar.
- **Selection** domain object (`domain/Selection.h`) exists; analyzers consume
  `Selection`, not `QRect`.
- **AnalyzerRegistry** exists with `HistogramAnalyzer` + peers registered.
- `ImageViewer` loads exclusively through `ImageRepository`; contains no
  `QImageReader`/`Decoder`/`CacheManager` decode calls.

## Acceptance (frozen — must stay green)

- [x] Open a directory and load any JPEG/PNG/BMP; `ImageRepository` returns a
      valid `ImageFrame` (no UI decode).
- [x] `ImageViewer` contains no `QImageReader`/`Decoder`/`CacheManager` decode
      calls (QWidget layer holds zero image-decoding logic).
- [x] TIFF listed; decode enabled once `qtiff` + `libtiff-6.dll` deployed.

- [x] Pixel Inspector reports RGB under cursor, read from `ImageFrame`.
- [x] `m3pipeline_tests` acceptance suite passes (27 checks; TIFF skips if
      codec absent).
- [x] `test_m3acceptance` proves the two P0 bars: `loadDirectoryAsync` ≤100 ms
      on 1000 images; `ThumbnailPipeline` first thumbnail ≤200 ms. 5/5 checks.

## Regression guard

Any future change that makes `ImageViewer` (or any `QWidget`) call a decoder,
`QImageReader`, or open a file directly is a **regression against this frozen
milestone** and must be rejected at review.
