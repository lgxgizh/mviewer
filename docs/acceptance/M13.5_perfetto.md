## Goal (review Phase 5)

> Stop guessing; measure Scan / Decode / Thumbnail / Render / Upload / Compare.
> Each pipeline stage has p50/p95/p99 from a **real trace** (not inference).

## What was built (additive — freeze-safe)

1. **`src/core/trace/TraceSink.h` + `TraceSink.cpp`** — a self-contained,
   zero-dependency span recorder. When `MVIEWER_ENABLE_PERFETTO` is defined,
   the existing `MV_TRACE_*` instrumentation points (already placed in the hot
   paths) forward spans here; `mviewer::trace::flush(path)` writes a **Chrome
   trace-format JSON** (openable in `ui.perfetto.dev` and `chrome://tracing`).

2. **`src/core/trace/Trace.h`** — replaced the prior broken enabled-branch
   (it `#include <perfetto/trace_event.h>`, which was never vendored) with a
   real forward to `TraceSink`. The no-op default branch is unchanged, so the
   **green build is untouched** (macros compile away).

3. **CMake** — `option(MVIEWER_ENABLE_PERFETTO ... OFF)` in `src/CMakeLists.txt`
   (default OFF → no new dependency, no behavior change in the canonical build).
   `TraceSink.cpp` added to `mviewer_core`.

4. **`mviewer_bench --trace <file>`** — flushes the trace at exit when built
   with the flag; otherwise prints a clear "needs MVIEWER_ENABLE_PERFETTO=ON"
   notice. No behavior change in the green build.

5. **`scripts/trace_report.py`** — parses the Chrome JSON and prints per-stage
   p50/p95/p99 (the RFC acceptance). stdlib-only.

## Why Chrome JSON, not binary `.perfetto-trace`

A real Perfetto trace requires vendoring the perfetto SDK (protobuf + large
build). To honor the AGENTS.md freeze (no new hard dependency in the green
build), the sink is fully self-contained — hand-written JSON, zero external
libraries. The timeline data (per-stage durations) is identical, and the
Perfetto web UI ingests Chrome JSON natively via its import flow.

## Honest scope note

`mviewer_bench` is a headless `QCoreApplication`, so it exercises the
**decode / repository** hot paths. The UI-path scopes (`RenderEngine::
executeCommand`, `ImageViewer::paint`, `ThumbnailCache::get/put`) are wired with
the identical `MV_TRACE_SCOPED` macro and WILL emit under any windowed context
built with the flag (e.g. `MViewer` itself, or `ui_screenshot` /
`product_browse_tests`). To capture them:

```
cmake -DMVIEWER_ENABLE_PERFETTO=ON ..   # separate build dir
mviewer_bench --trace ui.json           # or run MViewer and close
python scripts/trace_report.py ui.json
```
